# Public Project Map

## Batch 1: Public Foundation

### 1. `quant-research-portfolio`

Purpose:

- Serve as the portfolio landing page.
- Explain the research-to-production thesis.
- Map all public repositories.
- Record public safety boundaries.

### 2. `ares-public-architecture`

Purpose:

- Demonstrate a runnable synthetic quant lifecycle.
- Show data, signal, portfolio, risk, execution, reporting, and manifest layers.
- Provide tests and a toy backtest.

Public boundary:

- Synthetic data only.
- Toy signal only.
- Mock execution only.
- No real universe, live configuration, production parameters, or trading logs.

### 3. `vnq-datalake-public-manual`

Purpose:

- Document a professional data lake architecture for systematic research.
- Show schema design, universe snapshots, vendor mapping, update workflows, validation, and manifests.

Public boundary:

- Documentation and synthetic examples only.
- No paid vendor data, private cache, access material, or production paths.

## Batch 2: System Architecture Expansion

### 4. `chronos-master-portfolio-demo`

Purpose:

- Show master portfolio governance across multiple sleeves.
- Demonstrate drawdown governor logic and risk scaling using toy sleeve returns.

### 5. `quant-execution-audit-toolkit`

Purpose:

- Show order, fill, position, PnL, and manifest lifecycle.
- Use mock broker and dry-run execution only.

## Batch 3: Research Breadth Expansion

### 6. `vhedge-risk-allocation-demo`

Purpose:

- Show defensive multi-asset allocation concepts.
- Demonstrate volatility targeting and turnover-aware rebalancing.

### 7. `equity-alpha-research-lab`

Purpose:

- Show generic factor research framework.
- Include OOS validation and anti-lookahead checks.

### 8. `qinglong-ashare-research-lab`

Purpose:

- Show A-share market structure handling.
- Include suspension, limit-up/limit-down, tradability flags, and cohort-based toy research.
