# Public Boundary

## Public by Design

The public repositories may include:

- Synthetic data
- Toy signals
- Mock brokers
- Generic risk controls
- Generic execution lifecycle examples
- Documentation and diagrams
- Sample manifests
- Sample validation reports
- Unit tests

## Private by Design

The public repositories must not include:

- Proprietary alpha signals
- Real production parameters
- Real universe selection rules
- Live broker configuration
- Real broker identifiers
- Real order, fill, position, or PnL logs
- Paid vendor data
- Private data caches
- Private path names
- Production run manifests
- Trained model artifacts

## Language Standard

Public repositories should be written in English only.

## Performance Claims

Toy backtest output is only architecture evidence. It should not be described as a trading result or investment performance.
